# 钢材基差季节性看板

这个看板以 `/Users/liuziqi/Desktop/daily dashboard/历史数据1.xlsx` 作为业务数据母表，用本地 SQLite 作为运行缓存和查询加速层，并在浏览器里展示 05、10、01 合约基差季节性图。

当前现货口径包括：

- 板材：乐从热卷、上海热卷、唐山热卷
- 长材：普方坯、杭州中天、中天厂发、中心库、广州韶钢

当前主图固定展示四张基差季节性图：中心库、杭州中天、乐从热卷、上海热卷。

## 启动

安装依赖：

```bash
python3 -m pip install -r requirements.txt
```

启动看板：

```bash
python3 dashboard_app.py
```

打开：

```text
http://127.0.0.1:8765
```

## 数据口径

- 现货：Excel 中的 `上海热卷价格`、`乐从热卷价格`、`唐山热卷价格`、`中天厂发价格`、`中心库价格`、`广州韶钢价格`、`普方坯价格`、`杭州中天价格`
- 期货：热卷品种对应 `HC`，长材品种对应 `RB`
- 基差：`现货价 - 对应合约收盘价`
- 合约映射：对任意日期，05/10/01 分别映射到下一个尚未过去的对应月份合约，例如 `2026-06-30` 的 10 月合约为 `HC2610`，05 月合约为 `HC2705`
- 自动收盘价来源：优先使用新浪期货合约日 K，失败时回退到上期所日行情

## 数据维护

业务数据统一维护在 Excel：

- `现货价格`：宽表，列为 `日期`、各现货品种价格。
- `期货收盘`：明细表，列为 `日期`、`品种`、`合约月`、`合约代码`、`收盘价`、`来源`、`更新时间`。

网页保存现货、手动保存期货、自动获取期货收盘价时，都会先写入 Excel；Excel 写入成功后，再同步刷新 SQLite。每次写入 Excel 前会在同目录生成一个带时间戳的 `.bak` 备份文件。

如果 SQLite 丢失或需要刷新，启动看板或点击页面里的 `从 Excel 同步` 都会从 Excel 重建缓存。

## 文件

- `dashboard_app.py`：本地 Web 看板
- `data/steel_basis.sqlite`：本地 SQLite 缓存，包含从 Excel 同步来的现货和期货收盘价，以及接口缓存
- `scripts/backfill_target_contracts.py`：补齐热卷相关 HC 合约收盘价
- `scripts/backfill_rb_spot_contracts.py`：补齐长材相关 RB 合约收盘价
- `requirements.txt`：Python 依赖

## 备注

历史 Excel 最初只有现货价格，没有 HC/RB 各合约的完整历史收盘价。当前版本会把页面录入或接口获取到的期货收盘价写回 Excel 的 `期货收盘` sheet；季节性图只会绘制同一日期同时具备现货价和期货收盘价的数据。

日度更新时，右侧 `新增 / 更新日度数据` 区域可以在同一日期下一次性录入所有现货品种价格；留空的品种不会被覆盖。

当前版本适合单机或小范围私有协作的原型验证。正式多人协作时，建议将 Excel 母表放到共享网盘或迁移到云端数据库，并增加登录、权限和操作日志。
